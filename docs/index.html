Overview

local-search is a fully self-hosted web search and page-fetching pipeline designed for LLMs and AI agents. It wires together a metasearch engine, a headless browser scraper, and an anti-detect Firefox into a single Docker Compose stack, then exposes the whole thing via a clean HTTP gateway API and an MCP server.

No API keys required. No rate limits. No data leaving your machine.

SearXNG
localhost:8888

Self-hosted metasearch engine aggregating Google, Bing, DuckDuckGo, and dozens more. JSON API enabled.

Firecrawl
localhost:3002

Self-hosted scraping engine. Fetches pages and returns clean markdown. No auth required on self-hosted.

Camoufox
internal :3000

Anti-detect Firefox running Xvfb-backed virtual headless. Wired in as Firecrawl's default browser engine.

Gateway
localhost:8088

FastAPI service exposing /search and /fetch — the two endpoints LLMs actually call.

MCP server
stdio

Node.js stdio MCP server wrapping the gateway into web_search and fetch_page tools.

Quick start

Prerequisites

Docker and Docker Compose (v2). Node.js 18+ (for the MCP server only).

Start the stack

# First run pulls ~4 GB of images (firecrawl, playwright-service, searxng, …)
docker compose up -d

# Verify everything is healthy
curl http://localhost:8088/healthz
# → {"gateway":"ok","searxng":"ok","firecrawl":"ok"}

Run a search

curl 'http://localhost:8088/search?q=rust+async+runtime&max_results=5'

Fetch a page as markdown

curl -X POST http://localhost:8088/fetch \
  -H 'Content-Type: application/json' \
  -d '{"url": "https://example.com"}'

Register the MCP server with Claude Code

claude mcp add local-search -- node /path/to/local-search/mcp-server/index.js

Stop

docker compose down

Web console

The gateway ships a built-in web console at http://localhost:8088. Open it in a browser to search and fetch pages interactively without writing any curl commands.

Search tab

Full-text search with optional time range filter, result count control, and one-click scrape of top results into markdown.

Fetch tab

Paste any URL and get its content as rendered markdown. Toggle stealth mode to route through the Camoufox anti-detect browser.

The header shows a live health status for the gateway, SearXNG, and Firecrawl.

HTTP API

All endpoints are served by the FastAPI gateway at http://localhost:8088. POST endpoints accept JSON bodies; the GET /search variant accepts query parameters.

Health check

GET /healthz Liveness check and upstream reachability

Returns status for each upstream service. Use this to confirm the stack is fully running.

# Response
{
  "gateway": "ok",
  "searxng": "ok",
  "firecrawl": "ok"
}

Search

GET /search Query string params
POST /search JSON body

Parameters

FieldTypeDefaultDescription
query / qstringrequiredSearch query
max_resultsinteger 1–308Number of results to return
fetch_contentbooleanfalseScrape top results and include full page markdown
fetch_topinteger 1–103How many top results to scrape (when fetch_content is true)
categoriesstring"general"SearXNG category: general, news, it, science, …
languagestring"en"Language/locale code
time_rangestringnullday, week, month, or year

Examples

# Simple GET search
curl 'http://localhost:8088/search?q=llm+context+window&max_results=5'

# POST: search + scrape top 3 results into markdown (one call)
curl -X POST http://localhost:8088/search \
  -H 'Content-Type: application/json' \
  -d '{
    "query": "llm context window comparison",
    "max_results": 8,
    "fetch_content": true,
    "fetch_top": 3,
    "time_range": "month"
  }'

Response shape

{
  "query": "llm context window comparison",
  "result_count": 8,
  "results": [
    {
      "title": "...",
      "url": "https://...",
      "snippet": "...",
      "engine": "google cse",
      "score": 1.0,
      "published_date": "...",
      "content": "… full page markdown …"  // only when fetch_content=true
    }
  ]
}

Fetch page

POST /fetch Fetch a single URL and return it as clean markdown

Parameters

FieldTypeDefaultDescription
urlstringrequiredURL to fetch
only_main_contentbooleantrueStrip navigation, footers, and boilerplate
max_charsintegernullTruncate returned markdown to this many characters
stealthbooleanfalseForce rendering via Camoufox anti-detect browser (slower; use for bot-protected sites)

Examples

# Standard fetch
curl -X POST http://localhost:8088/fetch \
  -H 'Content-Type: application/json' \
  -d '{"url": "https://docs.example.com", "max_chars": 20000}'

# Stealth mode for bot-protected sites (Cloudflare, etc)
curl -X POST http://localhost:8088/fetch \
  -H 'Content-Type: application/json' \
  -d '{"url": "https://protected-site.com", "stealth": true}'

Response shape

{
  "url": "https://docs.example.com",
  "title": "Example Docs",
  "description": "…",
  "markdown": "# Example Docs\n…",
  "status_code": 200,
  "truncated": false   // present only when max_chars was hit
}
Note on stealth mode: even without stealth: true, Firecrawl only escalates to a browser engine when its plain HTTP fetch isn't sufficient. Setting stealth: true forces browser rendering (via Camoufox) every time, which is slower but much harder to block.

MCP server

A Node.js stdio MCP server ships in mcp-server/. It wraps the gateway's two endpoints into MCP tools that any MCP-compatible LLM client can call.

Register with Claude Code

claude mcp add local-search -- node /path/to/local-search/mcp-server/index.js

Generic MCP client config

{
  "mcpServers": {
    "local-search": {
      "command": "node",
      "args": ["/path/to/local-search/mcp-server/index.js"],
      "env": { "GATEWAY_URL": "http://localhost:8088" }
    }
  }
}

Direct access

Both upstream services are also reachable directly for debugging or advanced use.

SearXNG

http://localhost:8888 — full web UI and JSON API.

# JSON search
curl 'http://localhost:8888/search?q=example&format=json'

Firecrawl

http://localhost:3002 — raw Firecrawl API. Auth is disabled on self-hosted; pass any bearer token if a client requires one.

# Direct scrape
curl -X POST http://localhost:3002/v2/scrape \
  -H 'Authorization: Bearer local' \
  -H 'Content-Type: application/json' \
  -d '{"url": "https://example.com", "formats": ["markdown"]}'

# Firecrawl's own search (uses SearXNG internally)
curl -X POST http://localhost:3002/v2/search \
  -H 'Authorization: Bearer local' \
  -H 'Content-Type: application/json' \
  -d '{"query": "example query", "limit": 5}'

Configuration

Browser engine

Camoufox is wired in as Firecrawl's default browser engine via PLAYWRIGHT_MICROSERVICE_URL. To switch back to stock Playwright/Chromium: 

PLAYWRIGHT_MICROSERVICE_URL=http://playwright-service:3000/scrape docker compose up -d api

Or set it in a .env file at the repo root.

Resource limits

Default CPU/RAM limits per service in docker-compose.yml:

ServiceCPUsMemory
api (Firecrawl)4.04 GB
camoufox2.02 GB
playwright-service2.02 GB

Raise cpus and mem_limit in docker-compose.yml for heavier crawling workloads.

SearXNG engines

Active search engines are configured in searxng/settings.yml. By default the standard engine set is active. See the SearXNG docs for the full engine list.

Keep this private. searxng/settings.yml disables the bot limiter and enables the JSON output format — features that should only be exposed on localhost or a private network.

Architecture

All services run inside a single Docker Compose network (backend). Only the gateway and SearXNG expose ports to the host.

  LLM / agent ──MCP──▶  mcp-server (node, stdio)
                              │
                              ▼
  LLM / agent ──HTTP─▶  gateway :8088  (FastAPI)
                         │           │
                         ▼           ▼
                    /search       /fetch
                         │           │
                         ▼           ▼
                    SearXNG      Firecrawl api
                    :8888 (int)  :3002 (int)
                                     │
                             ┌───────┴───────┐
                             ▼               ▼
                         camoufox    playwright-service
                         (default)   (fallback/optional)
                             │
                      Redis + RabbitMQ + Postgres

Request flow — search with content enrichment

When fetch_content: true is set:

1. gateway  →  SearXNG     (JSON search results)
2. gateway  →  Firecrawl   (parallel scrape of top N URLs)
3. Firecrawl→  camoufox    (headless Firefox renders each page)
4. gateway  ←  Firecrawl   (clean markdown for each page)
5. caller   ←  gateway     (combined: results + markdown)

Request flow — stealth fetch

1. caller   →  gateway     (POST /fetch, stealth: true)
2. gateway  →  Firecrawl   (waitFor forces browser rendering)
3. Firecrawl→  camoufox    (anti-detect Firefox, Xvfb headless)
4. caller   ←  gateway     (markdown)
